# Peer-to-Peer Rental Marketplace

Student: Maria Lavigina | ID: 40683364

## Project overview

I built a mobile rental marketplace app called Library of Things as part of SET09102 coursework. The idea is simple - users can list items they own and rent from other people. Think of it like a library but for anything.

The app is built with .NET MAUI so the same code runs on Android, Windows and Mac. I wrote it in C# using .NET 10.0 and followed the MVVM pattern throughout. I tested and ran it on Android using an emulator.

### What the app can do

- Register and log in using JWT authentication
- Browse all available items with search and category filter
- Create, update and delete your own listings
- Full rental workflow: Request → Approve → Out for Rent → Returned → Completed
- Leave a review after a completed rental
- Find nearby items using GPS

## Setup

1. Open Docker Desktop first
2. Open the project in VS Code - click **Reopen in Container** when prompted
   - If no prompt: `Ctrl+Shift+P` → `Dev Containers: Reopen in Container`
3. Migrations are already applied - you don't need to run them unless cloning fresh

### Dependencies

The `.csproj` files handle everything. After any changes run `dotnet restore` - same idea as `npm install`.

Main packages:
- `CommunityToolkit.Mvvm` 8.4.0 - MVVM helpers
- `Npgsql.EntityFrameworkCore.PostgreSQL` 10.0.0 - PostgreSQL
- `BCrypt.Net-Next` 4.0.3 - password hashing

## How to run the app

**On your host machine:**
```bash
emulator -avd Pixel_9_Pro
adb -a -P 5037 nodaemon server start
```

**Inside the container:**
```bash
adb devices                    # check emulator is visible
dotnet clean -c Debug          # only needed after .xaml changes
dotnet build -c Debug
adb install -r StarterApp/bin/Debug/net10.0-android/android-x64/com.companyname.starterapp-Signed.apk
```

Tap the purple icon on the emulator to launch. The emulator uses its own keyboard.

## Architecture

Three projects:

| Project | What it does |
|---|---|
| `StarterApp/` | Mobile app - Views, ViewModels, Services |
| `StarterApp.Database/` | Models, Repositories, AppDbContext |
| `StarterApp.Migrations/` | Creates and updates database tables |

MVVM pattern: `Model (data) → ViewModel (logic) → View (UI)`

## Database migrations

Migrations are already applied. If you clone fresh and need to recreate them:

```bash
dotnet ef migrations add AddMarketplaceTables \
  --project StarterApp.Database \
  --startup-project StarterApp.Migrations

dotnet ef database update \
  --project StarterApp.Database \
  --startup-project StarterApp.Migrations
```

`AppDbContextModelSnapshot.cs` is auto-generated by EF Core - never edit it manually.
